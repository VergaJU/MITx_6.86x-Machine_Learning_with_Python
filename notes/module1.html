Hypotesis space

Hypothesis space is the set of all the possible legal hypothesis. This is the set from which the machine learning algorithm would determine the best possible (only one) which would best describe the target function or the outputs.

Lets say you have an unknown target function f:X→Y that you are trying to capture by learning. In order to capture the target function you have to come up with some hypotheses, or you may call it candidate models denoted by H h1,...,hn where h∈H. Here, H as the set of all candidate models is called hypothesis class or hypothesis space or hypothesis set.

Why large margin classification:

Many linear classifiers can correctly separate the points, what we need is the one that has the largest margins, so that clearly devide the points. Considering noisy points we can find in the future as contours of the given points in the training set, a large margin classifier still classify correctly the noisy points.

To optimize (maximising) the space of the margins margins, we can imagine other 2 decision boundaries parallel and equidistant to the decision boundary that push and change the original decision boundary to reduce the points misclassified by the boundaries. Seen as an optimization problem we have two terms:

  • Regolarization term: Disfavouring margin boundaries far from original decision boundaries:
  • Loss: Counterweight for the violation the points fall in the "wide" decision boundary

With lower values of CCC the model is more simplier. Now plotting the Loss (or errors) over the CCC parameter we can see that with the training set the loss tend to lower with the increasing of CCC. On the other hand, the test error, that is tipically higher than the test error, tends to reduce until a certain point and then increase again because the model is trained to recognize correctly only the training points. We can see a pont C∗C^*C∗ where we have the best generalizing point with low trainign error and the minimum of the test error. This devide the problem in 2 regimes:

  • Overfitting where C>C∗C>C^*C>C∗ with low training error and high test error

To find the value C∗C^*C∗ we have to simulate the test set, to do so we take the whole training set and devide in a main portion used to train the model and the smaller portion called validation set used to simulate the test loss and approximate the test error (validation error) to optimize the performance.
